# 🕵️ SQLNOIR: T-SQL Mystery Investigations

## 📌 Overview

This project is a hands-on, gamified exploration of **T-SQL (Transact-SQL)** through interactive detective-style investigations using the **SQLNOIR platform**.

Each mystery simulates a real-world data investigation where SQL queries are used to uncover clues, analyze patterns, and ultimately identify a culprit. The project emphasizes both **technical SQL proficiency** and **analytical problem-solving skills**.

---

## 🎯 Objectives

* Develop strong proficiency in **T-SQL**
* Apply advanced SQL concepts including:

  * JOINs
  * Subqueries
  * CTEs (Common Table Expressions)
  * Aggregations and filtering
* Strengthen logical reasoning through **step-by-step investigations**
* Simulate real-world data analysis and debugging workflows

---

## 🛠️ Tools & Technologies

* **SQLNOIR Interactive Platform** (primary environment)
* **T-SQL**
* **VSCode** (for SQL file organization and submission)

│── Mystery_003_TheMiamiMarinaMurder.sql
│── Mystery_004_TheVanishingDiamond.sql

Each file contains:

* Step-by-step investigation queries
* Inline comments explaining logic

---

## 🔍 Investigation Methodology

Every mystery follows a structured workflow:

1. **Crime Scene Analysis**
   Identify key details (location, date, description)

2. **Clue Extraction**
   Use SQL queries to filter and retrieve relevant evidence

3. **Data Linking (JOINs)**
   Combine multiple tables to build relationships

4. **Suspect Identification**
   Narrow down candidates using conditions and logic

5. **Verification**
   Confirm findings using transcripts, logs, or confessions

6. **Final Query (CTE)**
   Consolidate the entire investigation into a single optimized query
